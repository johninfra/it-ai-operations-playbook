# SOPs (Standard Operating Procedures)

## Overview

This folder contains Standard Operating Procedures (SOPs) designed to standardize common IT support, system administration, and user management tasks.

The purpose of these procedures is to ensure operational consistency, reduce errors, improve efficiency, and provide technicians with clear step by step guidance for performing routine support activities.

These SOPs reflect common workflows found within enterprise IT departments, help desks, managed service providers (MSPs), and corporate support environments.

# Standard Operating Procedures

## 1. Password Reset SOP

### Purpose

Provide a standardized process for securely resetting user passwords.

### Scope

Applicable to:

- Active Directory environments
- Enterprise support desks
- Internal user support

### Procedure Overview

1. Verify user identity
2. Locate user account
3. Reset password
4. Require password change at next login
5. Verify successful access
6. Update support ticket

### Benefits

- Improved security
- Consistent user verification
- Faster resolution
- Reduced support escalations

---

## 2. Account Lockout Resolution SOP

### Purpose

Provide a structured process for resolving locked user accounts.

### Scope

Used when users are unable to access company systems due to account lockouts.

### Procedure Overview

1. Verify user identity
2. Locate user account
3. Review lockout status
4. Unlock account
5. Investigate root cause
6. Verify successful login
7. Document findings

### Common Causes

- Incorrect password attempts
- Cached credentials
- Mobile device synchronization issues
- VPN authentication failures
- Expired passwords

### Benefits

- Faster restoration of access
- Improved troubleshooting
- Reduced downtime
- Better user experience

---

## 3. User Account Provisioning SOP

### Purpose

Provide a standardized process for creating and provisioning new user accounts.

### Scope

Used during employee onboarding and access provisioning.

### Procedure Overview

1. Receive approved onboarding request
2. Create Active Directory account
3. Configure user attributes
4. Assign Organizational Unit
5. Add security groups
6. Configure email access
7. Validate account functionality
8. Document completion

### Provisioning Tasks

- User account creation
- Group assignment
- Department assignment
- Email configuration
- Access permissions
- Documentation updates

### Benefits

- Consistent onboarding
- Improved security
- Faster provisioning
- Reduced administrative effort

---

## 4. VPN Configuration SOP

### Purpose

Provide a standardized process for configuring VPN access for remote users.

### Scope

Used when deploying secure remote access solutions.

### Procedure Overview

1. Verify user authorization
2. Install VPN client
3. Configure VPN profile
4. Configure authentication settings
5. Test connectivity
6. Verify access to internal resources
7. Document configuration

### Validation Steps

Verify:

- Successful VPN connection
- Internal resource access
- DNS resolution
- Authentication functionality
- MFA functionality

### Benefits

- Secure remote access
- Consistent deployments
- Reduced connectivity issues
- Improved user productivity

---

## 5. Laptop Deployment Procedure SOP

### Purpose

Provide a standardized process for preparing and deploying company laptops.

### Scope

Used for:

- New employee onboarding
- Hardware refreshes
- Device replacements

### Procedure Overview

1. Verify hardware inventory
2. Install operating system
3. Apply updates
4. Join domain
5. Install required software
6. Configure security settings
7. Verify functionality

9. Update asset inventory

### Configuration Tasks

- Windows deployment
- Domain join
- Software installation
- Security configuration
- User profile setup
- Asset tracking

### Benefits

- Consistent deployments
- Improved security
- Reduced setup time
- Better asset management
